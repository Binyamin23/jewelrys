Jewelrys api documentation

Routes

its all http methods , you can use them for your request.

  • Sign Up: /users/signUp (POST)

    • name (required)

    • email (required)

    • password (required)

    • role

  • Sign In: /users/signIn (POST)

    • name (required)

    • password (required)

  • User's Info: /users/myInfo (GET)

    • auth-token: (required)
      • its only for register users , you must send auth-token through header


  • UserList Info: /jewelrys/myList (GET)

    • auth-token: (required)
      • its only for register users , you must send auth-token through header

    • Get 10 products by first page: /jewelrys (GET)

    • Get 10 products by page number: /jewelrys?page=number (GET)

    • Get X products in each page: /jewelrys?perPage=number (GET)

    • Sort products: /jewelrys?sort=sort-key (GET)

    • Reverse by sort key: /jewelrys?reverse=yes (GET)

    • Search products by name or info: /jewelrys/search?s=word-key (GET)

    • Price range: /jewelrys/prices?min=number&max=number (GET)

    • Get products by category: /jewelrys/category/category-name (GET)

  • Add product: /jewelrys (POST)

    • auth-token: (required)
      • its only for register users , you must send auth-token through header

    • name (required)

    • info (required)

    • category (required)

    • img_url

    • price (required)

  • Edit product: /jewelrys/id(jewelry) (PUT)

    • auth-token: (required)
      • its only for register users , you must send auth-token through header

    • name (required)

    • info (required)

    • category (required)

    • img_url

    • price (required)

  • Delete product: /jewelrys/id(jewelry) (DELETE)

    • auth-token: (required)
      • its only for register users , you must send auth-token through header
